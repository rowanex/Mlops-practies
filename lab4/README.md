# Лабораторная работа №4: Версионирование данных с помощью DVC

Данная работа демостнирует возможности утилиты **DVC (Data Version Control)** для управления жизненным циклом данных в рамках MLOps-цикла. В ходе работы был создан автоматизированный конвейер обработки данных с хранением промежуточных версий в облаке.

## 📁 Структура проекта
```
├── lab4/                # Основная рабочая директория
├── .dvc/                # Конфигурация DVC
│   ├── data/            # Директория с данными (под контролем DVC)
│   │   └── .gitignore   # Исключает физические CSV из Git
│   ├── dowload_data.py  # Скрипт загрузки (Titanic dataset)
│   ├── process_data.py  # Скрипт фильтрации признаков
│   ├── fill_na.py       # Скрипт обработки пропусков (Age)
│   ├── one_hot.py       # Скрипт One-Hot Encoding (Sex)
│   ├── dvc.yaml         # Описание стадий Pipeline (DAG)
│   └── dvc.lock         # Фиксация состояний пайплайна
└── README.md
```

## 🛠 Настроенный функционал

### 1. Удаленное хранилище
Настроено удаленное хранилище на **Google Drive**. Все версии данных после обработки выгружаются в облако командой:
```bash
dvc push
```

### 2. Версионирование датасета
В ходе работы были созданы и зафиксированы три основные версии данных:
1.  **Начальная:** Выбор признаков `Pclass`, `Sex`, `Age`.
2.  **Очищенная:** Заполнение пропусков в поле `Age` средним значением.
3.  **Финальная:** Применение One-Hot Encoding для категориального признака `Sex`.

### 3. Автоматизация (DAG Pipeline)
Вместо ручного добавления файлов (`dvc add`), настроен пайплайн через `dvc.yaml`. Это позволило построить направленный ациклический граф зависимостей (**DAG**).

## Инструкции по воспроизведению

### Переключение между версиями
Чтобы вернуться к любой точке истории (например, к версии с пропусками), выполните:
1. Найдите хеш коммита: `git log --oneline`
2. Переключите код: `git checkout <hash>`
3. Синхронизируйте данные: `dvc checkout`

### Запуск конвейера
Благодаря настроенному DAG, полное воспроизведение всех этапов обработки данных выполняется одной командой:
```bash
dvc repro
```
DVC автоматически определит, какие скрипты изменились, и перезапустит только необходимые этапы.
